# Atelier 2.1.0 — project home, git, and a real command palette

The biggest update since the rebrand. Atelier now has a per-project home,
built-in git, a working slash-command palette, and a proper "add project" flow —
plus a long list of daily-use fixes.

## Project home

- **Clicking a project opens its home**, not a blank screen — a per-project
  dashboard with a stat strip (sessions · tokens), a grouped **sessions list**
  (Today / Yesterday / This week), and a **Changes** tab.
- **Sessions are titled by what they were about** — the first real prompt of the
  conversation, not the last "/clear". Open one to resume it (with a loading
  state, no more freeze on big sessions).
- The session row **is** the start button: click it and the session starts —
  resumed if it can be, fresh if its history is gone, with the real reason shown
  if it can't open.

## Git, built in

- A **Changes** tab with real `git`: staged/unstaged files, live diffs with
  add/delete coloring, **stage / unstage**, and **commit** — no terminal needed.
- **Ask Claude to write the commit message** from your staged diff.
- The project home shows your branch and ahead/behind at a glance.

## Slash commands

- Type **`/`** for a command palette. `/clear`, `/cost`, `/model`,
  `/permissions`, and `/help` run *in the app*; `/init`, `/review`, `/compact`
  send a prompt.
- **Your own commands appear** — anything under `.claude/commands` (project) or
  `~/.claude/commands` (global), with `$ARGUMENTS` support and namespacing.
- A chosen command locks into a **chip** so you only type its arguments.

## Add a project

- A prominent **Add project** button: **open a folder** or **clone a repo**
  (real `git clone` with progress), with a "detected" panel that reads the
  folder's git status, `CLAUDE.md`, and stack.

## Context, models, and polish

- A **context-window meter** in the composer shows how full the conversation is
  (e.g. `62% · 124k/200k`) so you know when to `/clear` or `/compact`. Window
  sizes come from Anthropic's own spec, not hardcoded guesses.
- **Chrome-style tabs** that truncate long titles instead of stretching.
- **Projects are ordered by recent activity** — what you touched last is on top.
- **`+ New session` just starts** — no second "Start session" button.
- Full-width chat and lists; instant click feedback; bigger click targets on
  rows and dropdowns; model/turn errors are shown instead of silently dropped.

## Under the hood

A focused bug sweep hardened the new code: the streaming message no longer
rebuilds on every token (text selection works mid-stream), git can't deadlock on
a chatty repo, resuming is race-safe, and failed turns surface their real error.
