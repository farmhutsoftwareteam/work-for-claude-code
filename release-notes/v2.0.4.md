# Atelier 2.0.4 — permissions that stick, full state visibility, retry

A working-session pass focused on the things that made daily use frustrating.

## Permissions

- **Default is now `accept edits`** — claude can edit files without a prompt
  per edit, while still asking for riskier tools. Dial up to `bypass` or down
  to `plan` / `default` whenever you want.
- **Your choice sticks.** Changing the permission mode now persists across new
  tabs, restarts, and resumed sessions — it no longer silently resets to
  per-call prompting every spawn.
- **Change it mid-conversation** — the header menu (and Shift+Tab in the
  composer) applies the new mode to the live session immediately *and* saves
  it as the default for next time.
- Mode list cleaned to the four claude actually honors, ordered least → most
  permissive: `plan → default → accept edits → bypass`, each with a one-line
  description of what it does.

## See all of claude's states

- Claude emits a stream of `system` events — stop-hook output, informational
  notices, connection-retry errors, and more. We used to surface only three
  and silently drop the rest. Now they appear as subtle inline notes in the
  transcript.
- **Connection errors are visible.** `api_error` events (claude auto-retrying
  a flaky connection) now show a "retrying (n/N)" banner instead of looking
  like a hung session.

## No more dead air after sending

- Sending a message shows an immediate **"Working…"** cue and auto-scrolls to
  it — no more "nothing happens, then Thinking shows up late." The transcript
  now follows streaming replies all the way down, and the stale result footer
  from the previous turn is cleared the instant a new turn starts.

## Retry

- A **retry** button on the result footer re-sends your last message without
  retyping it.

## Still rough

State notes are a first pass — some render their raw body and aren't yet
designed per-type. Functional visibility now; nicer formatting to come.
