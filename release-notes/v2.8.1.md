# Atelier 2.8.1 — a real crash fixed, plus a full stability pass

- **Fixed a crash in the co-driven terminal.** Running something like
  `vim`, `htop`, or `tmux` co-driven for long enough could crash the app
  — a ring-buffer bug that was reachable automatically, with zero agent
  activity, once enough output had passed through the pane.
- **Two real security gaps closed in the skill-from-repo flow.** A
  malicious skill name in a cloned repo could have written files outside
  Atelier's skills folder; a crafted URL could have been read as a git
  flag instead of a repo address. Both are now blocked before they reach
  git or the filesystem.
- **Quitting Atelier now actually stops your chat sessions.** Previously
  only terminal tabs were signaled on quit — an active chat tab's
  `claude` process could be left running in the background.
- **The ACP preview no longer leaks its process** when you close it.
- A long list of smaller fixes across streaming, the composer, skills,
  agents, and usage tracking — stuck "working" states, silently dropped
  errors, memory that grew over long sessions, and a few UI papercuts.
  Nothing user-facing changes; this release is entirely under-the-hood
  stability work.
