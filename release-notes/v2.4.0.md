# Atelier 2.4.0 — the co-driven terminal

Interactive commands used to be where agent automation died: `eas submit`,
logins, installers — anything that asks questions — would hang in Claude's
Bash tool. Now Claude runs them in a real terminal you both drive.

## Co-driven terminal

- **Claude runs interactive commands in a visible terminal pane** — a real
  PTY, so CLIs that ask questions actually work. Claude reads the screen and
  answers routine prompts itself ("Which build? [1]"); anything it can't
  answer, it asks you in chat.
- **You can type in the same pane at any time** — click in and drive.
- **Secrets never touch the model.** When a password prompt appears, the pane
  flips to a clay-bordered *secure prompt* state: Claude's reads are withheld
  and its typing is rejected — you type directly, hidden from the agent. This
  is enforced in the engine, not just the UI.
- **Everything is attributable**: an INPUT strip shows exactly what the agent
  typed, a dovetail mark pulses while it's watching the screen, and the header
  carries live elapsed / exit status.
- When Claude reaches for an interactive command via plain Bash, the
  permission card offers **Run co-driven** — one click reroutes it into the
  shared terminal.

## Model selection, finished

- **Your model choice sticks.** Picking a model now also sets the default for
  new sessions — no more silent reverts to Opus 4.8.
- **The picker works everywhere, always current**: the model list comes from
  your Claude Code binary itself (new models appear the day the CLI ships
  them), is remembered across launches, and the picker now works even before
  a session starts. Add Project's model menu shows proper names too.
