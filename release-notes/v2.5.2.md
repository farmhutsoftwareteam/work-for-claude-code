# Atelier 2.5.2 — MCP sign-in that heals itself

If an MCP server ever showed "Unrecognized client_id" in your browser, or a
sign-in silently timed out — this release is for you.

- **Sign-in now self-heals.** Providers quietly expire the OAuth client
  registration cached on your machine; every sign-in after that opened a
  perfectly-formed link the provider rejected. Atelier now resets that state
  automatically on every sign-in — one click, fresh registration, working
  consent page. No terminal commands, no retries.
- **claude.ai connectors finally sign in properly.** Connectors (Gmail,
  Calendar, Drive…) authorize on claude.ai itself — Atelier now opens the
  right page (it previously never opened the browser for these) and says
  plainly: finish authorizing there, and it connects on your next session.
  No more false red errors after a correct grant.
- **Real errors, fast.** A server the CLI doesn't recognize used to mean a
  silent 3-minute wait and a generic failure; you now get the CLI's actual
  message within seconds.
- Hardening: the hidden sign-in terminal is allocated extra-wide so long
  authorization URLs can never wrap mid-link.
