# Atelier 2.0.2 — the three things that made v2 hard to use

Three blocking UX bugs from the first day of dogfooding v2, all fixed.

## Transcript scrolls during streaming

The view only scrolled when a *new* message was added — but a reply
streams token-by-token into the same message, so the count never
changed mid-answer and the transcript sat still while text ran off the
bottom. Now it pins to the bottom as the reply streams in (and through
the result footer / loading state), and jumps to the bottom when you
open a resumed session.

## New chats stop "replacing" the current tab

The tab strip was filtered to the rail's selected project, so starting
a chat in a different project made your previous tab disappear from the
strip — it looked like the new chat had replaced the old one. The strip
is now global: every open chat is a visible tab (like browser tabs),
scrollable, with a small project label on each when you've got chats
open across more than one project. Opening a session from history also
moves the rail's project selection to match.

## Permission requests are a real modal now

When Claude asks to use a tool it doesn't have standing access to, the
request used to be a small inline card between the transcript and the
composer — easy to scroll past, easy to miss entirely. It's now a
window-level modal: the whole window dims and the request sits front and
centre with Approve / Deny until you resolve it.
