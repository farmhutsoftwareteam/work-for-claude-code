# Atelier 2.0.3 — making v2 actually usable for daily work

Six chat-surface bugs from the first sessions of dogfooding v2 — all the
kind that bite within minutes of real use.

## Transcript scrolls during streaming

The view only scrolled when a *new* message was added — but a reply
streams token-by-token into the same message, so the transcript sat
still while text ran off the bottom. Now it pins to the bottom as the
reply streams in (through the result footer and loading state), and
jumps to the bottom when you open a resumed session.

## New chats stop "replacing" the current tab

The tab strip was filtered to the rail's selected project, so starting a
chat in a different project made your previous tab vanish from the strip
— it looked like the new chat had replaced the old one. The strip is now
global: every open chat is a visible tab (like browser tabs), scrollable,
with a small project label on each when you've got chats open across more
than one project.

## Permission requests are a real modal

When Claude asks to use a tool it doesn't have standing access to, the
request used to be a small inline card that was easy to scroll past. It's
now a window-level modal — the whole window dims and the request sits
front and centre with Approve / Deny until you resolve it.

## Each tab keeps its own draft, attachments, and scroll position

The composer and transcript weren't keyed per-tab, so a half-typed
message, pending attachments, and the scroll position from one chat bled
into others when you switched tabs. Each tab now keeps its own state.

## Enter no longer fires a second turn mid-stream

Pressing Enter while a reply was still streaming used to send another
user turn into the same session, interleaving with the in-flight stream.
You can keep typing your next message while it works; it sends once the
turn finishes.

## "esc to interrupt" actually works now

The composer always advertised it, but the escape key was never wired —
only the Stop button worked. Escape now interrupts the running turn.
