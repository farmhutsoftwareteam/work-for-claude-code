# Atelier 2.9.5 — MCP setup keeps getting safer

- **Can't save a broken server anymore.** Saving an MCP server with an
  empty credential field (like a Supabase access token you meant to
  fill in) is now blocked with a clear message — previously it saved a
  server that read "configured" but could never authenticate.
- **Plugin MCP servers are now visible and reusable.** Servers that
  ship inside plugins (like the official Supabase plugin's OAuth
  server) now appear in the MCP panel before a session starts, and get
  the same "→ project" action as your own servers — copy one into a
  project and bind it with a URL parameter
  (`?project_ref=…`), keeping OAuth and writing zero tokens to disk.
- **Buttons across the MCP editor and marketplace now use Atelier's
  own design language** instead of native macOS styles.
