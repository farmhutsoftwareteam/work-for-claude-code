# Atelier 2.11.0 — it's Atelier everywhere now, and your sessions travel

- The download finally says Atelier. The disk image, the mounted volume, and the app you drag to Applications are all named Atelier — not "Work". Updates keep working exactly as before: the app's internal identity is unchanged, so existing installs update seamlessly (if you installed before this release, the file on disk may keep its old name until you re-download — cosmetic only).
- Move a conversation to another Mac: File → Export Session (⇧⌘E) writes a portable .ateliersession file; Import Session (⇧⌘I) on the other machine files it under the right project there and opens it. Works for both Claude and Codex. Credentials and unsent drafts never leave your Mac. Importing refuses to silently overwrite an existing copy of the same session, and asks before replacing.
- Fixed a real path bug underneath that feature: Atelier now derives Claude's session-storage folder the way Claude actually does (every non-alphanumeric character, not just slashes), so projects with spaces, dots, or underscores in their paths resolve first try — and CLAUDE_CONFIG_DIR is honoured if you've relocated your session storage.
- Codex multi-agent work is now visible: delegation cards with live status and elapsed time, a persistent strip that keeps running agents in view after their spawn scrolls away, and click-to-peek into each agent's own thread. Agents are named ("audit_measure_backend", "main thread") instead of raw thread IDs, fan-outs show every agent, and a re-spawned agent's card comes back to life.
- A hardening pass over all of the above before shipping: 34 fixes, including several message-loss paths (a message typed into a resting tab whose sign-in had expired is handed back instead of being silently replayed later as a real turn), atomic export/import writes, and a settled agent's final report never being overwritten by later chatter.
