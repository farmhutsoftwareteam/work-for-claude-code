# Atelier 2.10.1 — provider continuity you can trust

- Fixed Codex turns failing immediately after a Claude-to-Codex handoff because Atelier sent Codex's `additionalContext` field in the wrong protocol shape.
- Switching between Claude and Codex now retains both native session identities, drafts, and the visible working timeline instead of replacing the conversation you came from.
- Relaunch restores both provider slots and safely falls back to the retained provider if the previously selected runtime or history file is unavailable.
- Claude and Codex now share the same composer chrome, attachment controls, send/stop behavior, context meter, and responsive layout.
- Persistent CLAUDE and CODEX badges use distinct text, geometry, accessible labels, and restrained provider colors in tabs, session controls, composers, overflow search, and History.
- The unified History rail now discovers closed Codex threads through the local app-server and resumes them alongside Claude sessions without starting a model turn.
- Codex thread startup and history restore no longer wait for MCP inventory; repeated MCP refresh notifications are coalesced while configuration and OAuth remain provider-owned.
- Codex working tabs now show the correct elapsed timer, and long streamed transcripts avoid a full indexed-array allocation on every refresh.
