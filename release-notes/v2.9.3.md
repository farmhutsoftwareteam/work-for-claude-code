# Atelier 2.9.3 — MCP setup, actually easy this time

- **The MCP marketplace is finally connected.** "+ add" now opens a
  searchable catalog with one-click, prefilled install instead of a
  blank form asking you to already know the exact command or URL.
- **Use an existing server in a new project.** Any global MCP server
  now has a "→ project" action that copies it into the current
  project with the same name — point Supabase (or anything else) at
  this project's own resources without duplicating the whole config
  by hand.
- **OAuth sign-in now starts automatically** the moment you add a
  remote server — no more saving, watching it silently fail, and
  hunting for a sign-in button that didn't exist yet. Every OAuth
  server also gets a "Sign in again…" option for switching accounts
  or recovering from a stale token.
- **Fixed: MCP servers stuck on "starting" forever.** Status used to
  be a one-time snapshot from when the session first spawned — a
  server that was still connecting then would show "starting" for
  the rest of the session, even long after it actually connected.
  Status now refreshes live.
- **A guardrail against committing secrets.** Saving a literal API
  key or token into a project-shared server config now offers a
  one-tap "keep it private" alternative instead of silently writing
  it into a file meant for git.
- Servers for the same service under different accounts (e.g. three
  Linear workspaces) now group under one heading instead of listing
  as unrelated rows.
