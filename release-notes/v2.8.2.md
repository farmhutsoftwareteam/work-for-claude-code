# Atelier 2.8.2 — the tab bugs, found and fixed

- **Fixed: switching to a tab sometimes showed an empty chat.** The
  transcript's tab-switch handling was keyed on a memory address that
  macOS can silently recycle between sessions — when it did, the reset
  that positions the conversation never fired and the tab rendered
  nothing. Sessions now carry a stable identity, so this can't happen.
- **Fixed: reopening the app with several tabs bounced the scroll
  around.** Each restoring tab was re-rendering the active conversation
  mid-settle; the restore burst now collapses into a single update.
- **Fixed: thinking blocks were always empty.** The expand arrow opened
  onto nothing because the streamed thinking text was never captured —
  it now accumulates exactly like reply text does.
- **Select across paragraphs.** Drag-selection used to die at the edge
  of a single paragraph or bullet; prose now renders as continuous,
  fully selectable sections. You can also right-click any message →
  Copy message.
- **Running commands show elapsed time.** A tool call still working
  after a few seconds now shows a live counter — a five-minute test run
  no longer looks identical to a hung one. Reopened conversations also
  stop showing forever-spinning tool rows.
